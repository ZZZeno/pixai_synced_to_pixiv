# PixAI → Pixiv / 𝕏 Publisher

PixAIの作品をワンクリックでPixiv・𝕏（Twitter）に同期投稿するChrome拡張機能です。

## ✨ 機能

### PixAI側
- **作品詳細ページ**: `[+ Pixiv]` `[+ 𝕏]` ボタンで作品をキューに追加
- **作品一覧ページ**: プロフィール・検索結果・探索ページなどで、各作品カードにホバーで `P` / `𝕏` ボタンが表示
- GraphQL API経由で作品データ（画像・タイトル・タグ・説明文）を自動取得
- 無限スクロールで読み込まれた新しいカードにも自動でボタンを追加

### Pixiv投稿
- イラスト投稿ページに自動入力：画像・タイトル・キャプション・タグ・年齢制限・AI生成フラグ
- 複数作品の一括投稿対応（キャプションは作品ごとに空行で区切り）
- タグ処理：スペース含むタグは除外、複数作品時は共通タグのみ、`PixAI`タグを自動追加（最大10個）
- WebP → PNG自動変換

### 𝕏（Twitter）投稿
- ツイート作成ページに自動入力：テキスト（タイトル＋PixAIリンク＋ハッシュタグ）＋画像
- ハッシュタグ検証：特殊文字（`：` `/` `!`など）を含むタグは自動除外
- クリップボード貼り付け → ファイル入力 → ドラッグ＆ドロップの3段階フォールバック
- SPA遷移対応（MutationObserver + URL監視）

### キュー管理
- Pixiv / 𝕏 独立した2つのキュー
- ポップアップUIでタブ切り替え、個別に追加・削除・並べ替え・一括削除
- バッジに両キューの合計数を表示

## 📦 インストール

1. このリポジトリをクローン
   ```bash
   git clone https://github.com/ZZZeno/pixai_synced_to_pixiv.git
   ```

2. Chromeで `chrome://extensions/` を開く

3. 「デベロッパーモード」を有効化（右上トグル）

4. 「パッケージ化されていない拡張機能を読み込む」→ クローンしたフォルダを選択

## 🚀 使い方

### 作品をキューに追加

**作品詳細ページ（`/artwork/ID`）：**
- ページ右側に表示される `[+ Pixiv]` / `[+ 𝕏]` ボタンをクリック

**作品一覧ページ（プロフィール・検索など）：**
- 作品カードにマウスを合わせると右上に `P` / `𝕏` ボタンが表示
- クリックで直接キューに追加（詳細ページを開く必要なし）

### Pixivに投稿

1. キューに作品を追加済みの状態で [Pixivイラスト投稿ページ](https://www.pixiv.net/illustration/create) を開く
2. 自動的にフォームが入力される
3. 内容を確認して投稿

### 𝕏に投稿

1. キューに作品を追加済みの状態で [𝕏作成ページ](https://x.com/compose/post) を開く
2. 自動的にツイート内容と画像が入力される
3. 内容を確認してポスト

## ⚙️ 設定・仕様

| 項目 | 設定値 |
|------|--------|
| 年齢制限 | 常に「全年齢」 |
| 性的表現 | 常に「無」 |
| AI生成作品 | 常に「はい」 |
| タグ上限（Pixiv） | 10個 |
| キャプション形式 | タイトル + PixAIソースURL |
| 画像形式変換 | WebP → PNG（自動） |

## 🏗️ 技術構成

```
manifest.json          # Manifest V3 拡張設定
background.js          # Service Worker（キュー管理・画像変換）
pixai-content.js/css   # PixAI側コンテンツスクリプト（データ取得・ボタン表示）
pixiv-content.js/css   # Pixiv側コンテンツスクリプト（フォーム自動入力）
twitter-content.js/css # 𝕏側コンテンツスクリプト（ツイート自動入力）
popup.html/js          # ポップアップUI（キュー管理）
icons/                 # 拡張アイコン
```

### データ取得
- PixAI GraphQL API（Persisted Query）で作品メタデータを取得
- DOM fallback：API取得失敗時はページDOMから直接抽出

### ストレージ
- `chrome.storage.local` に `pixivQueue` / `twitterQueue` として保存
- 各キューアイテム：画像URL・タイトル・タグ・説明文・PixAIリンク

## 📝 注意事項

- PixAIのログイン状態が必要です（GraphQL APIにアクセスするため）
- Pixivの投稿ページはDOMセレクタに依存しているため、Pixiv側のUI変更で動作しなくなる可能性があります
- 𝕏の画像アップロードはクリップボード貼り付け方式のため、環境によって動作が異なる場合があります
- PixAI GraphQLのPersisted Query Hashは将来変更される可能性があります
